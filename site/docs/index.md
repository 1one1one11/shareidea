# PB Insight

PB는 고객의 자산 목적에 맞는 전략을 함께 설계하고 실행을 돕는 파트너입니다.

저를 더 많이 이용해주시기 바랍니다.
이 공간은 증권사 PB 사용법과 실제 운영 아이디어, 그리고 저의 투자관을 정리하는 문서 허브입니다.

## 홈페이지 안내

아래 링크에서 원하는 주제로 이동할 수 있습니다.

### 1-1. 현재 PB 티어 시스템

- 문서: [pb-tier-system](ai-presentation/pb-tier-system.md)
- 설명: 고객 티어 기준, 서비스 차등, 응대 SLA를 정리합니다.

### 1-2. PB's Pick

- 문서: [PB's Pick](ai-presentation/pbs-pick.md)
- 설명: 제가 보는 섹터/종목/상품 아이디어와 투자 판단 기준을 정리합니다.

### 1-3. PB's Idea

- 문서: [PB's Idea](ai-presentation/pbs-idea.md)
- 설명: 확률분포 기반 운용, 리서치 자동화, 상담 운영 아이디어를 공유합니다.

### 1-4. PB Insight Style

- 문서: [pb-insight-style](ai-presentation/pb-insight-style.md)
- 설명: 앞으로 문서를 어떻게 개선할지 기준을 공개합니다.

### 1-5. 주간 점검 체크리스트

- 문서: [weekly-report](ai-presentation/pb-weekly-report.md)
- 설명: 매주 반복 가능한 운영 점검 항목을 제공합니다.

## 나의 작업 스타일(향후 반영 규칙)

- 1차: 판단보다 실행 프레임을 먼저 만든다.
- 2차: 감정보다 확률과 근거를 우선한다.
- 3차: 문서는 `표 + 체크리스트 + 다음 액션`을 기본 단위로 구성한다.
- 4차: 좋은 문장보다 재현 가능한 과정이 더 중요하다.

## 내 투자관 요약

- 마켓 타이밍보다 `시장 온도 측정`을 우선한다.
- `산다/판다` 이분법보다 `확률분포 기반 포지션 사이징`을 사용한다.
- 높은 ROE, 낮은 부채, 높은 현금전환의 `퀄리티`를 선호한다.
- 외부 변수 공포로 매도하지 않고, 가설이 깨졌을 때만 매도한다.
- 섹터/기업 분석은 `PESTEL + 15문항 점검`으로 구조화한다.

## 새 문서 추가

1. `site/docs` 아래에 `.md` 파일을 추가합니다.
2. 내부 링크는 `.md`로 작성해도 빌드 시 `.html`로 자동 변환됩니다.
3. 상단 메뉴 노출이 필요하면 `config/site.json`의 `navigation`에 등록합니다.
