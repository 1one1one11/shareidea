# pb-insight 스타일

이 페이지는 PB Insight 문서 체계를 통일하기 위한 기준입니다.

## 1) 문서 쓰기 원칙

- 결론은 문서 상단, 근거는 중간, 액션은 마지막에 배치
- 표/점수표/체크리스트를 기본 구성요소로 둔다.
- 감정보다 가설, 반증, 종료조건을 먼저 기록한다.

## 2) 문서 템플릿(강제)

### 제목

- 목적, 기간, 대상이 1줄에 보이도록 작성

### 본문 핵심 블록

1. 현재 진단(숫자)
2. 판단 근거(표)
3. 리스크와 반증 조건
4. 다음 액션(체크리스트)

## 3) 수치 기준

- 성과는 추적 가능 지표(점수, 비율, 기한)로만 측정
- 문서 간 비교를 위해 동일 포맷 사용

## 4) 투자관 반영 규칙

- 마켓 타이밍보다 시장 온도 측정을 우선
- 기대수익률은 확률범위로 제시
- 매도는 가설 붕괴 조건 충족 시 우선 실행

## 5) 새 페이지 점검 체크리스트

- [ ] 링크가 `.md` 형식으로 작성되어 있는가?
- [ ] 표/체크리스트/액션이 있는가?
- [ ] 수치 기준(등급/임계값)이 명시되었는가?
- [ ] 다음 액션의 완료기한이 있는가?

## 6) 작성 마감 규칙

- 최종 문구는 단정형보다 실행형 문장으로 마무리
- 추정은 근거 주석과 함께 표기
